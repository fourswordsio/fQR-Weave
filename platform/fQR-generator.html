About fQR Weave Generator:

fQR Weave uses arweave.org blockchain to create fQRs from transaction's ID (already explained in fQR Weave page). Generator's source code is public and available on github.

In short, we already published a fQR code generator prototype which is available for anyone interested into testing the platform (alpha version). But later on, fQRs generators will be limited to registred (verified products owner) only.

How To Integrate fQR Into Your Business ?

Within the first stable version's release, products/stores owners will be able to apply from the official fQR Weave website for a verified wallet account. After the verification process, our team will build, for each verified user, a generator instance compatible with his business and needs (shoes store, supermarket, certificates...).

After passing the verification & evaluation process, the user has to pay a subscription fee for using this service, which will cover the technical support, generator coding, and more custom needs. AR coin will be the main acceptable cryptocurrency in fQR Weave platform: special offers for paying in AR. More cryptocurrencies payments options (coins/tokens) may be added later on to increase the adoption with ease.

More informations about verification critica and subscription fees will be published in the near future.

What If A Verified User Lost His Wallet ?

With the beta version release, we will add more improvements to the system, specially for fQR Reader & Generator codes, which will have a "fQR destructuring" function. This functionality will be activated and executed after the owner's request

Hence, every fQR code generated by the lost owner wallet will be destroyed, but even that, the original transaction data will still accessible by CLI (or any interaction with arweave blockchain). Surely that's possible because blockchain is immutable.
